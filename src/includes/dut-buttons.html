Notes:

  • A Cancel button SHALL not be used unless the the form uses a multi-step process.
  • A Cancel or Close button should clear the contents of the form for security/privacy.
  • A Cancel button should always be used in a Multi-step or multiple screen process so the User feels safe that they are aborting a process.
  • When used, the Cancel button SHALL always appear to the right of the Submit button and appear as a link or Alternative button.
  • In a multi-step or multiple screen process, Back and Next SHALL always be used, and Submit SHALL always conclude the process and form submission.
  • A Reset or Clear button SHALL not be used on a form, because studies indicate a much higher percentage of errors.

Which Label to use?

vs.

Use a Submit button when the User is anonymous. Use a Save button when the User is logged into a system and will be updating data within a system.

vs.

Always use a Search button when there is a query involved. Even though you are submitting a form while performing a search, it is more intuitive to use Search as the button label.

vs. vs.

When returning to a previous screen or step, use Back as the button label. When ceasing a process, such as updating your profile, use Cancel as the button label. Exit SHALL not be used.

vs. vs.

When data or a state in the application has changed, it is recommended to use Refresh as a label. The Reload lable implies getting more data, while Refresh can be used for multiple instances, including reloading of data, or refreshing of a screen visual. Reset SHALL not be used as it implies the clearing or deleting of data.
